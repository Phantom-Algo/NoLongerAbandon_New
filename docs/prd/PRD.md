# NoLongerAbandon - 基于 AI 的本地英语单词学习 & 记背助手

## 技术栈

采用前后端分离范式 + RESTful API

### 前端
1. Vue3 + Vite + TS

### 后端
1. Web应用层 SpringBoot3
2. AI业务层 Spring AI
3. 数据存储层 sqlite

## 需求

### 鉴权模块
因为是本地应用所以不需要鉴权模块

### 基于英语单词搜索的模块
1. 用户输入单词后，后端搭建的AI工作流会根据输入的单词输出包含多个预设模块（例如“基本释义”“音标”“例句”“记背方法”）的内容（markdown格式），形成一张单词卡
    1. AI输出内容必须要严格包含要求的预设模块（通过工作流实现严格约束）
    2. 用户可以手动添加预设模块（模块主题 + 模块内容）
    3. 输出完单词卡后，当前输出的内容会存储在数据库中，下次用户搜索用一个单词就不需要再通过AI进行生成，而是直接复用
    4. 用户可以要求AI重新生成整个单词卡或某一个模块（允许加上自己的需求，但AI必须按照预设模块进行生成）（该功能实现与普通的与AI chat不同，这个是一个单独的功能）
    5. 用户可以在光标划定某一行或几行或几个字之后右键，点击“选择当前上下文”，该上下文就会注入，用户在与AI进行chat交互时能够基于注入的上下文进行交流
    6. 生成的单词卡可以加入到用户创建的单词本中

### 基于中文的搜索模块
1. 用户输入完中文后，会先根据中文找出几个合适的英文单词并给出释义（也由AI完成），供用户选择，用户选择后后续工作与“基于英语单词搜索的模块”需求一样。

### 单词本模块
1. 用户可以创建多个不同的单词本（不可重名）
2. 用户可以将搜索的单词加入到单词本中（一个单词可以加入到多个单词本中）
3. 点击单词本后展示的默认按照加入时间排序的单词列表，支持分页查询；点击某个单词项后进入到对应单词卡中；此外还有对单词项的基础CRUD；除了根据时间排序，还支持根据单词的“搜索次数”排序，这就需要实现对单词搜索次数的记录。
4. 点击进入到单词卡中后————与“基于英语单词搜索的模块”差不多

### 关于AI生成模块
！！！注意，AI 生成单词内容与AI chat是两个不同的功能
1. 单词卡的生成需要按照预设的模式进行生成，基于AI工作流（可以考虑采用plan-execute模式，一个plan Agent对需求进行分析，然后发出多个Execute Agent根据 plan agent的要求完成各自模块的生成，最后汇总审查，没问题就输出）；单个模式的生成也差不多思路
2. 关于 AI chat
    1. 要求支持流式输出
    2. 要求支持创建不同的会话
    3. 要求支持选择模型

### 设置模块
1. 用户可以在这里添加自定义的模型（输入API KEY + Base url + model name），此后AI chat可以选择自定义的模型
2. 用户在这里设定AI 生成单词内容的模型
3. 用户可以在这里设定自己的个人信息，以及是否允许AI读取自己的个人信息（若允许，则支持AI在最开始接收输入阶段注入用户个人信息）

## 代码要求

### 前端
1. 前端要遵循Vue3开发良好的架构范式
2. 前端UI使用ElementUI，不允许使用奇怪的蓝紫渐变色，图标必须使用svg
3. 需要设置LOGO（LOGO图片在static/img/LOGO.png）

#### 颜色
1. 主题色：#008858
2. 功能色：
    1. 错误色：#d54941
    2. 警告色：#e37318
    3. 成功色：#56c08d
3. 中性色：#fff

### 后端
1. 遵循controller-service-mapper范式
2. 使用 Mybatis Plus进行数据库访问
3. mapper代码不能够侵入到service中，这是铁律！！！
4. 在用户第一次启动服务时，需要检查数据库表是否创建，若尚未创建则需要进行初始化操作
