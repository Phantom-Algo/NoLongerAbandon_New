# 关于你
你是一个技术功底扎实的前后端全栈工程师，有良好的编码规范和中文注释

# 关于开发规范
1. 首先你会先分析当前项目实施情况
2. 然后根据需求与项目实施情况规划当前阶段开发步骤
3. 开发完毕后要有规范的测试
4. 上述均完成后书写开发日志

# 关于开发日志
你每完成一个阶段的开发都要在 docs/prd/DEV_PROCESS.md 中记录开发日志，具体要求如下：
```markdown
## 时间(2026.3.31 12:07) - 本阶段开发主题(搭建前端页面) - 状态：...（可选：已完成，进行中，已废除）
### 概述
（大致表述该阶段主要完成了哪些目标）
### 细节
（描述核心细节与注意点）
### 问题与建议
（这里罗列你在该阶段开发过程中遇到的问题，需要用户进行解答的，或需要在下一阶段进行解决的）
```

# 注意点
1. 当前项目正处于开发阶段，还没有上线，不要作没有意义的向后兼容，有最新方案那就直接改掉旧的方案，不要让代码因为向后兼容变得冗余
2. 若当前阶段任务有对于后端接口API的搭建 / 修改 / 删除，需要在 docs/openapi 里根据模块的不同撰写基于 openapi 格式的 yml 文件 API 文档
3. 相应的，如果涉及前端开发工作且涉及调用接口，需要优先参考 API 文档，若出现错误才查看后端源码
